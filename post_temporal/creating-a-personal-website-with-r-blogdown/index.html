Creating a personal website with R-Blogdown

May 11, 2018 5 min read R, Website, Blogdown, R-Ladies, Hugo

This blog post contains a tutorial to create a personal website using HUGO (a static website generator), RStudio and the R Blogdown package. This is the tutorial followed in the R-Ladies Leuven May Meetup.

This tutorial is accompanied by an introductory presentation, available here (link not yet available).

In this example, we will work with the “Academic” template, but many others are available. You can check them all and choose your favorite from here

Prerequisites

  • R (preferrably >3.2.2, this tutorial was tested in v3.4.2)

  • Rstudio (this tutorial was tested in v1.0.153)

  • R Blogdown package

# Install Blogdown package
install.packages("blogdown")
  • HUGO: can be installed from within the Blogdown package, which makes our lives easier
# Install HUGO
blogdown::install_hugo()

  • Git and Github. To create a GitHub free account visit https://github.com/ and to download Git visit https://git-scm.com/

  • Very little (R-)Markdown knowledge. But you can always check the cheatsheet here. What you’ll need today is summarized in section 3. Markdown and maybe also 5. Embed code if you wish to add some R code to your blog posts.

Disclaimer

This tutorial shows the path I used to create my own personal website using Blogdown. I based the tutorial on my own experience and in the problems I found on my way. Also, I made some choices that were convenient for me, such as hosting the website on Github. Other posibilities (Netlify) exist, but are not covered in this tutorial, and guides for this can be found elsewere (TODO: provide links?)

Lets get started!

First step is to create a new R project to generate the website. Depending on your RStudio version you can create an “empty” project, or a website. Let’s use the first approach which is compatible with older RStudio versions.

Why do we want an R-project? Because it provides very useful features. Some examples:

  • The Addins tab on top. Click on it and you get a lot of functionalities that are specific for websites. For example, “New post” creates and formats the files required to add a new post to the blog. “Serve site” calls Hugo internally so that the website is generated. Once we click on “Serve site”, every time we save any file within the website project, the website is generated again on-the-go! Fantastic, isn’t it? In the lower-right pane Viewer you can explore how your website looks like.

  • It organizes the complete file structure for your website, so that you don’t have to worry about its architecture. With a few clicks you can have your website painlessly!

To create a new R Project go to File > New Project. Select to create your project in a New directory and then as an Empty project. Now you have to specify the directory name (which is the name of your project). You can use something as “test_website”. You also need to specify where the project directory will be stored (I chose “~/Documents/”).

Then, download the Hugo Academic template and generate your website. This template is a good choice for people in academia, as this website can be used to present your CV with your experience, degrees and, most importantly, publications. You can also add blog entries, such as this one. After this tutorial, you can try other Hugo templates and choose the one that best fits your needs.

All this can be done just by typing the following command in the Console of your newly created project:

# Generate a new website
blogdown::new_site(theme = "gcushen/hugo-academic")

Congratulations! You have created a website.

You have installed Blogdown, Hugo, and have created an R-project for a website with the Academic template. All this with only some mouse clicks and 3 lines of code in the Console.

First, you will realize that in RStudio, in the right pane Viewer, you can now see the website you just generated. This is the default template, we will work on modifying the different parts throughout this tutorial.

Take some time to check what has happened in the directory you created for this project. Several files and sub-directories have appeared.

Here I will explain some of the more relevant for the rest of this tutorial:

Content

The true content of your website lies here. Within this directory, there are specific sub-folders for the different sections of your website. For instance, when you create a new blog post later in this tutorial, it will be stored in the post subdirectory.

Public

This folder contains the rendered contents of your website in html. You don’t need to modify the contents of this folder, they will be automatically generated by Hugo and Blogdown. Later in this tutorial, we’ll show how to deploy the website using GitHub. To do so, we will need to make a GitHub repository for this directory. But just not yet.

Static

This folder contains some static files, such as pictures, that are used in your website. It is convenient to place them here and then reference them from your posts.

Themes

Contains the defaults for the theme of your website, in this case Academic. It also contains all the data from the example website that you just generated

Your website needs metadata!

You can go to Addins > Serve site and visualize the website you just created. The website that has been generated is just the template for the Academic theme. So now it’s time to personalize it!
